# UX Rules

## Purpose
These rules define how Rvdobuilds should behave as a calm studio homepage.
The site must feel clear and intentional on mobile first, then scale up cleanly.

## Core principles
1. The homepage must explain the studio fast.
2. Products must outrank tools.
3. Every section must justify its height.
4. Visual calm beats feature count.
5. Repetition is only acceptable when it strengthens hierarchy.

## Homepage rules
### Hero
- The hero must explain what Rvdobuilds is within one quick scan.
- Keep the headline strong and direct.
- Keep supporting copy short.
- Use one clear primary CTA.
- Add a secondary CTA only if it genuinely helps.

### Featured section
- Feature only the most important current product by default.
- Do not repeat the same product again immediately in a weaker form.
- If a featured product also appears later in Products, the later card must still add value through different framing or reduced emphasis.

### Products section
- Products must be easy to scan on mobile.
- Product cards must feel consistent.
- Use the same CTA pattern across all active products.
- "Coming soon" items should still look intentional, not disabled or forgotten.

### Tools section
- Tools must visually sit below Products in hierarchy.
- Tools should not compete with the featured product.
- Keep the section smaller and lighter than Products.

### About section
- Keep About short and useful.
- Focus on point of view, not biography.
- Avoid long personal story blocks.

## Navigation rules
- Keep navigation labels short and literal.
- Use Products before Tools.
- Keep the header compact.
- If the header is sticky, it must remain visually quiet.
- Do not let header chrome dominate the page.

## Hierarchy rules
- There must always be a clear most-important element in the viewport.
- Headlines, descriptions, metadata, and CTAs must not compete equally.
- Secondary text must stay readable, not too dim.
- Labels and eyebrows should support rhythm, not create clutter.

## CTA rules
- Each card gets one clear primary action.
- Do not mix button styles arbitrarily.
- If one section uses bordered dark buttons, adjacent sections should not switch to plain text CTAs without reason.
- Primary CTA text must stay explicit.

## Card rules
- Cards must feel substantial enough to carry real products.
- Avoid empty oversized cards.
- Avoid unnecessary nested cards inside already strong outer cards.
- If a card contains only simple text and one action, keep it visually efficient.

## Mobile rules
- Design for mobile first.
- Keep important content within a clear single-column rhythm.
- Avoid horizontal clipping of segmented controls, tabs, or phase rails.
- Avoid tiny touch targets.
- Never rely on hover to reveal critical actions.

## Spacing rules
- Use clear separation between major sections.
- Use tighter spacing inside cards than between cards.
- Reduce dead zones that do not strengthen the layout.
- Do not stack large empty spaces back-to-back.

## Copy rules
- Be direct.
- Be short.
- Avoid generic marketing filler.
- Avoid overclaiming.
- Prefer plain confidence over hype.

## Consistency rules
- The same type of content should use the same visual pattern.
- Product cards should look like one family.
- Tool cards should look like one family.
- Status labels should follow one shared logic.
- Button treatment should not drift section by section.

## Anti-patterns
Do not introduce:
- duplicate content that weakens hierarchy
- decorative sections that add no information
- bright accents across many elements
- app-dashboard chrome on a studio homepage
- overlong About sections
- random visual exceptions just to make one block look different
