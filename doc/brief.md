# Brief

## Project
Rvdobuilds.com

## One-line definition
A calm, premium studio site for useful software products and tools built to stay simple.

## Core positioning
Rvdobuilds is not a generic personal portfolio and not a flashy agency site.
It is a focused studio layer that presents a small set of products, tools, and experiments with clarity and restraint.

The site should communicate:
- useful software
- clear thinking
- premium restraint
- small, intentional scope
- trust through taste and focus

## Goal
Turn the current site into a sharper studio homepage that:
- clearly explains what Rvdobuilds is
- highlights the best current products first
- gives each product enough weight to feel real
- surfaces tools and workflow assets without competing with products
- feels visually aligned with Launchframe's dark premium system
- avoids utility-app chrome and avoids generic portfolio tropes

## Non-goals
Do not make this feel like:
- a developer docs site
- a dashboard or app shell
- a content-heavy blog
- a startup marketing site full of hype
- an agency selling custom services

## Primary audience
People discovering the work for the first time.
They should quickly understand:
- this is a solo software studio
- the products are real and intentional
- the design taste is calm and premium
- there is a coherent point of view behind the work

## Secondary audience
People who already know one product and want to understand the broader body of work.
For them, the site should act as a clean hub.

## Brand characteristics
The site should feel:
- calm
- deliberate
- lean
- premium
- modern
- technically credible
- visually restrained

The site should not feel:
- loud
- playful
- trendy for its own sake
- over-designed
- over-explained
- cluttered

## Visual direction
Use Launchframe as the base design language, but adapt it for a studio homepage.
That means:
- keep the same dark neutral-ink foundation
- keep the same warm text and quiet cool-blue accent
- keep subtle borders and layered surfaces
- reduce app-like chrome compared to ClaudeCodeKit
- allow a bit more openness and breathing room than Launchframe itself
- keep cards and sections disciplined rather than decorative

## Content priorities
Priority order:
1. Hero: what Rvdobuilds is
2. Featured product or products
3. Product collection
4. Tool collection
5. Short about / philosophy
6. External links only where meaningful

## Information hierarchy
The hierarchy should be:
- Studio identity first
- Products second
- Tools third
- About last

Products should always feel more important than tools.
Tools are supporting proof of thinking and process, not the main commercial layer.

## Key UX principles
- mobile-first
- fast to understand
- low-noise surfaces
- strong visual hierarchy
- consistent CTA patterns
- no redundant repeated content blocks unless there is a clear reason
- each section must earn its height

## Design outcome
A visitor should leave with this impression:
"This is a small software studio with taste, discipline, and real products."

## Success criteria
The redesign is successful if:
- the hero feels strong within one screen on mobile
- the site immediately reads as a software studio, not a personal résumé
- the visual system clearly matches Launchframe's color family
- products feel more prominent and credible than they do now
- the page no longer repeats or dilutes product emphasis unnecessarily
- CTA treatment is consistent across hero, featured, products, and tools
